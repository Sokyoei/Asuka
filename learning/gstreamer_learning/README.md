# GStreamer

## 核心基础概念

1. Element 元素/插件，独立的功能模块（如解码器、画图工具等），是最小的执行单元。
2. Pad 端口，模块之间的数据接口，负责传递数据。
    1. Sink Pad，输入端口（接收上游数据）。
    2. Source Pad，输出端口（向下游发送数据）。
3. Bin 容器，多个 Element 组成的集合，把一连串关联的功能模块封装成一个整体。

Elelement 有三大类型

| 类型            | 作用                                                | 端口                                           |
| :-------------- | :-------------------------------------------------- | :--------------------------------------------- |
| Source 源元素   | 产生数据流（rtsp 拉流、读取本地文件、摄像头读流等） | 只有输出端口（Source Pad），无输入端口         |
| Filter 处理元素 | 加工处理数据（解码、格式转换、画图、流合并等）      | 同时有输入（Sink Pad）和输出（Source Pad）端口 |
| Sink 终点元素   | 消费最终数据（播放画面、写入文件、推流等）          | 只有输入端口（Sink Pad），无输出端口           |

# 查看某个元素的所有信息
